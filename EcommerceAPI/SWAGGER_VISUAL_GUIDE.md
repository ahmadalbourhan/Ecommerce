## ?? Color Coding in Swagger UI

### HTTP Methods
```
?? GET      - Retrieve data (Safe, Idempotent)
?? POST     - Create data (Not Safe, Not Idempotent)
?? PUT      - Update data (Idempotent)
?? DELETE   - Remove data (Idempotent)
```

### Response Codes
```
?? 200-299  - Success (Green)
?? 300-399  - Redirect (Orange)
?? 400-499  - Client Error (Yellow)
?? 500+     - Server Error (Red)
```

## ?? Quick Testing Workflow

### Step 1: Create a User
```
1. Navigate to POST /api/users
2. Click "Try it out"
3. Enter user data
4. Click "Execute"
5. Note the returned user ID
```

### Step 2: Create a Role
```
1. Navigate to POST /api/roles
2. Click "Try it out"
3. Enter role name
4. Click "Execute"
5. Note the returned role ID
```

### Step 3: Assign Role to User
```
1. Navigate to POST /api/users/{id}/roles
2. Click "Try it out"
3. Set user ID from Step 1
4. Enter role ID from Step 2
5. Click "Execute"
```

### Step 4: Create a Product
```
1. Navigate to POST /product
2. Click "Try it out"
3. Enter product details with userId from Step 1
4. Click "Execute"
```
